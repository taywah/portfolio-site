Cursor Agent Skill Ecosystem

Production Cursor Agent Skills that turned recurring analyst workflows into governed, self-serve natural-language tools for business partners.

Role: Sole skill author for a sales-focused data science team Stack: Cursor + MCP, BigQuery, Markdown skill docs, Python skill helpers Period: 2026 Q1 — Q2

Problem

Business partners had recurring data needs that funneled into Data Science: market sizing, prospect pulls, manager-review workbooks, standardized exports, snapshot publishing, and data-quality checks. Data Science was the bottleneck on every one-off.

Worse: each request needed translation from business language into warehouse queries, scrub gates, and output formatting. That translation is doable, but it does not scale if every workflow depends on a data scientist.

Approach

Author Cursor Skills — SKILL.md files with scoped tool access, project context, and skill-specific data rules. Each skill is invokable by natural language and runs a governed workflow inside the user's Cursor session.

flowchart LR
  User["Business user asks for a governed data workflow"] --> Cursor[Cursor agent]
  Cursor --> SkillDetect[Skill detection]
  SkillDetect --> WB[Workflow-specific SKILL.md]
  WB --> BQMCP[BQ MCP: run sizing + scrub queries]
  WB --> Helpers[Python skill helpers: export + validation]
  WB --> Rules[Workspace rules: data-quality guardrails]
  BQMCP --> Output[Sized cohort + standardized export]
  Helpers --> Output
  Rules --> Output

Skills shipped

  • Market prioritization skill — context, metric definitions, and analysis rules for industry whitespace and planning workflows.
  • Reactivation workflow skill — end-to-end sizing, exclusion gates, contact export, and data-quality checks for dormant-account motions.
  • Play-sizing skill — classifies plays by intent, technology, industry, or combined themes and runs the correct sizing workflow.
  • Standardized export skill — applies persona validation, ranking, enrichment, and schema checks before handoff.
  • Manager-review skills — package review files, collect keep/dismiss decisions, and distribute cleaned outputs.
  • Critical-pass skill — self-audit checklist for stakeholder-bound data deliverables.
  • Insight-to-action skill — forcing function for turning analytical findings into leadership-ready recommendations.

Skill anatomy

Each skill is a single Markdown file with frontmatter and a structured body. Example structure:

---
name: governed-workflow
description: Run a recurring sales analytics workflow end-to-end ...
---

# Governed Workflow Skill

## Triggers
The user asks for a recurring sizing, export, or review workflow.

## Inputs required
- Segment or audience definition
- Review and handoff target
- Output destination

## Steps
1. Run sizing query from governed workflow context
2. Apply exclusion gates and activity scrubs
3. Run data-quality validation helpers
4. Generate standardized export
5. Run critical-pass self-audit

## Related
- workflow reference docs
- reusable validation helpers
- data-quality guardrail rules

Impact

  • Business partners could self-serve routine market sizing after onboarding.
  • Manager review cycles were codified end-to-end rather than recreated manually for every request.
  • The skill-authoring pattern was documented and adopted by a peer.
  • Recognized internally as an AI-first practitioner.

Lesson

The right abstraction for AI tooling at work is the workflow, not the model. Business partners do not want a chatbot; they want a trusted recurring workflow to run quickly, consistently, and with the right guardrails. Encoding expectations, scrub gates, and output formats frees the data scientist for higher-leverage modeling.
