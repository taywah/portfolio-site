NAICS Whitespace Explorer

Flask dashboard + static export ranking 309 industry segments on market size, reachable audience, win-rate signal, sales coverage, and outlier-deal potential. Turned vertical prioritization into a repeatable planning framework.

Role: Sole designer / builder Stack: Flask, BigQuery, pandas, Plotly, static export Period: 2025 Q3 — 2026 Q2

Problem

The sales team had a manually curated industry list that was static, hard to defend, and disconnected from observed pipeline behavior. Leadership needed a clearer way to identify high-opportunity and under-covered segments without relying on anecdotal prioritization.

Approach

Expand the scoring universe from a hand-picked list to a broad set of pipeline-active industry codes. Score each segment on a consistent metric set. Define whitespace mechanically using low coverage plus meaningful reachable audience. Detect outlier-deal potential via Tukey IQR so segments with high-value wins get surfaced even when medians are modest.

What I built

  • 309-segment industry universe built from won opportunities, open pipeline, and outbound activity rather than a 43-segment hand-curated list.
  • Whitespace flag: segments with below-median sales coverage and 100+ reachable prospects to support action.
  • Outlier-deal detection: Tukey's IQR fence flags segments with unusually large wins while avoiding overreaction to sparse data.
  • Revenue fallback logic: layered estimates keep sparse segments visible without letting one deal dominate the ranking.
  • Sales-cohort filtering: enforced in SQL so every metric reflects the same business motion.
  • Static stakeholder export: made the dashboard shareable with leaders who did not need direct warehouse access.
  • Sub-industry aggregation: related industry codes roll up cleanly with counts summed and rates weighted.

Code excerpt — Tukey outlier detection

# For each industry segment with 3+ wins, compute Tukey IQR fence
# and flag any deal above Q3 + 1.5*IQR as an outlier.

def detect_outlier_deals(deal_amounts):
    if len(deal_amounts) < 3:
        # Fallback to global 95th percentile for sparse segments
        return [d for d in deal_amounts if d > GLOBAL_P95]
    q1 = np.percentile(deal_amounts, 25)
    q3 = np.percentile(deal_amounts, 75)
    iqr = q3 - q1
    threshold = q3 + 1.5 * iqr
    return [d for d in deal_amounts if d > threshold]

Impact

  • Produced an executive-ready 6-vertical prioritization recommendation grounded in market size, reachability, sales coverage, revenue density, attach behavior, and outlier-deal potential.
  • Added quantified vertical readouts such as 488 outlier-deal flags and $924K expected revenue in one retail segment, 411 outlier-deal flags in live entertainment, $2,255 average revenue in SaaS, 22.8% transactional intent in financial services, and $2,391 revenue density per account in information.
  • Created a reusable scoring layer that downstream audience-building and planning workflows could consume, including a 38.32% SOM/SAM planning ratio.
  • Received director-level recognition for turning a complex prioritization problem into a clear stakeholder walkthrough.

What I'd do differently

I'd build the dashboard data layer as a single materialized table from day 1 instead of running separate per-metric queries on each request. The static export solved stakeholder access, but a single fact table would reduce the roughly 12-second page load and make the metrics easier for downstream tools to reuse.
