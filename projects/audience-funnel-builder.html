Audience Funnel Builder

A Streamlit + Vertex AI dashboard that lets sales partners define an audience, run a governed market-sizing and persona-validation workflow, and export a 77-column standardized sales-ready file with natural-language firmographic recommendations.

Role: Sole designer / builder / operator Stack: Streamlit, BigQuery, Vertex AI / Gemini, GCP, Plotly Period: 2025 Q4 — 2026 Q2

Problem

A sales organization had a recurring need: define a target audience, compare it to high-value customer patterns, remove already-covered accounts, validate likely buyer personas, and export the results in a format operations teams could upload cleanly.

The existing process was spreadsheet-driven and analyst-dependent. Each pull required manual translation, inconsistent formatting, and a turnaround cycle that kept Data Science in the middle of every audience request.

Approach

Build a Streamlit dashboard that runs a 3-stage BigQuery workflow, calls Vertex AI for natural-language parsing of the ask, surfaces a side-by-side firmographic comparison against a customer benchmark, and emits a standardized export directly.

flowchart LR
  User[User picks ICP / industry / filters] --> Streamlit[Streamlit UI]
  Streamlit --> BQ1[BQ Stage 1: TAM]
  BQ1 --> BQ2[BQ Stage 2: Existing-coverage scrub]
  BQ2 --> BQ3[BQ Stage 3: Persona validation]
  BQ3 --> Compare[Compare vs customer benchmark]
  Compare --> Gemini[Vertex AI: parse intent + recommend filters]
  Gemini --> Streamlit
  Streamlit --> Export[77-column sales export]

What I built

  • BigQuery pipeline: 3-stage workflow for market sizing, existing-coverage removal, and persona validation using governed source tables and reusable filters.
  • Vertex AI integration: natural-language parsing converts a business request into structured targeting inputs, then summarizes firmographic differences and filter recommendations.
  • Plotly comparison views: side-by-side distribution of prospect cohort vs customer benchmark across revenue, employee count, industry, and geography.
  • 77-column standardized sales export: generated a clean, operations-ready file so users did not need to manually reformat every audience pull.
  • Multipage Streamlit app: overview, pipeline builder, and customer-profile views for both self-serve execution and QA.
  • Productionization path: documented deployment, access, and data-refresh considerations for an internal app.

def recommend_filters(user_request, cohort_profile, customer_benchmark):
    structured_request = parse_request(user_request)
    comparison = compare_distributions(cohort_profile, customer_benchmark)
    return generate_recommendation(structured_request, comparison)

Impact

  • Shipped 3 iterations across Q1-Q2 2026, adding new targeting filters, improved fit scoring, and clearer auditability for audience pulls.
  • Created a shared operating channel for business partners, reducing ad hoc meetings and clarifying request intake.
  • Mentored an operations partner through the workflow so they could self-serve routine audience pulls instead of routing every request through Data Science.
  • Received director-level recognition for making a previously analyst-dependent workflow visible and usable for the sales organization.

What I'd do differently

If I were building v2 from scratch I'd separate the data pipeline from the Streamlit layer earlier and materialize the most expensive persona filters on a schedule. The first version optimized for speed to stakeholder feedback; the next version should optimize for runtime consistency and easier downstream reuse.
